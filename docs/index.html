Abstract

2026年4月21日、Google LabsがAIコーディングエージェント向けデザイン仕様フォーマット「DESIGN.md」をApache 2.0ライセンスでOSS化。YAMLフロントマター（機械可読なデザイントークン）とMarkdown本文（設計根拠）の2層構造により、Claude Code・Cursor・Copilotなど複数AIエージェントへのブランド規則配布を単一ソースで実現する。CLIツール（lint/diff/export/spec）により、WCAG準拠チェックやTailwind・W3C DTCG形式への変換をCI組み込みで自動化可能。仕様はalpha段階だが、W3C Design Token Format（DTCG）との互換設計により既存エコシステムとの接続性を確保している。

Insight

Abstract

AIエージェントが自律的に判断・実行する時代において、従来の「責任者の特定（RACI等）」だけでは事故後の責任の帰属が曖昧になるという問題を指摘した論考。筆者は「責任経路工学（Responsibility Pathway Engineering）」という概念を提唱し、Decision Owner・Approval Gate・Stop Authority・Repair Owner等7つの最小部品から成るResponsibility Pathway Layerを設計対象として定義している。具体的なサンプルデータや実証数値は含まれず、理論的・設計論的なフレームワーク提示が主眼。ISO/IEC 42001との接続は次稿以降に委ねられており、本稿は体系の入口に位置する。

Insight

SMU（南メソジスト大学）のMeuret・Ritz両博士とUCLAのCraske博士が、10年超の臨床試験を経てJAMA Network Openに発表したPositive Affect Treatment（PAT）は、うつ病患者の90%が抱えるアンヘドニア（快感消失）を直接標的とする15セッションの心理療法である。重篤なアンヘドニア・うつ・不安を持つ成人98名のRCTにおいて、PATは従来の陰性感情標的療法を全般的臨床状態の改善で上回り、1ヶ月後のフォローアップでも優位性を維持した。特筆すべきは、セッション中に陰性感情を一切扱わないにもかかわらず、うつ・不安症状の有意な改善が得られた「逆説的治癒」効果であり、脳の報酬系（報酬予期・学習・体験）の再訓練が中核メカニズムと結論付けられている。

Insight
「苦しみを取り除けば回復する」という精神医療の前提を根底から覆す発見であり、脳の報酬系と脅威系が独立して機能する神経科学的知見をそのまま治療設計に落とし込んだ点が革新的である。従来の認知行動療法（CBT）が「問題の除去」に集中するのに対し、PATは「ポジティブ資源の積み上げ」という全く異なる介入軸を確立したことで、慢性うつ・自殺リスク低減という最も難解な領域に新たな打ち手を生み出した。この知見は個人の心理療法にとどまらず、職場の活力設計（サヴァリング・感謝実践等のエクササイズのオンボーディングへの組み込み）や、Well-being計測における「ポジティブ感情量」を陰性感情の逆数ではなく独立指標として扱う組織診断の再設計にも波及しうる。
